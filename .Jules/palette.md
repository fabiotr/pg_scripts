## 2025-05-15 - Improving reporting hierarchy and documentation polish
**Learning:** Polishing report headers and documentation not only improves professionalism but also ensures a consistent user experience during database audits. Moving shared section headers (like '## Roles') out of version-specific logic ensures a stable document hierarchy regardless of the database version being audited.
**Action:** Always verify that shared report components have a consistent parent heading that is visible across all supported environment versions.
